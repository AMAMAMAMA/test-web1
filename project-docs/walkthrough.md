Google認証の実装に加え、ブログ機能全体の動作確認と改善を行い、Firebase Hostingへの公開まで完了しました。現在は独自ドメイン `deep-audit.tokyo` の反映待ちです。

現在、以下の2つのURLでアクセス可能です（独自ドメインは反映待ち）。

1.  **独自ドメイン**: **[https://deep-audit.tokyo](https://deep-audit.tokyo)**
    *   ※反映まで最大24時間程度かかる場合があります。
    *   ※SSL証明書の発行も自動で行われますが、少し時間がかかります。

    *   こちらは現在すぐに利用可能です。

## 実装済みの機能

1.  **Google認証**: `Sign in with Google` ボタンでログインできます。
2.  **記事の閲覧 (Public)**:
    -   トップページ (`/`) で「公開済み」の記事一覧を誰でも閲覧できます。
    -   記事詳細ページ (`/article/:id`) で記事の全文を読めます。
3.  **管理機能 (Admin Only)**:
    -   **ダッシュボード**: ログイン後、`/admin` で記事の管理（作成・編集・削除）ができます。
    -   **編集ボタン**: 管理者が記事詳細ページを閲覧している場合、直接「Edit Article」ボタンが表示され、編集画面へ移動できます。
4.  **安全性**:
    -   本番環境用のFirestoreルールに対応しました（認証済みユーザーのみ書き込み許可）。
    -   日付表示の不具合（作成直後のクラッシュ）を修正しました。

## 使い方（公開環境）

### 1. サイトへのアクセス
上記の公開URLをクリックしてください。

### 2. 管理者としてログイン
1.  ナビゲーションバーの「Login」をクリックします。
2.  「Sign in with Google」をクリックしてログインします。
    -   ※初回ログイン時、Googleの認証画面が表示されます。
3.  ログインに成功すると、自動的に管理ダッシュボード (`/admin`) にリダイレクトされます。

### 3. 記事の投稿・編集
ローカル環境と同じ手順で、記事の作成や編集が可能です。データはクラウド（Firestore）に保存されているため、どのURLからアクセスしても同じデータが見られます。

## 開発・修正フロー

今後、修正を行う場合は以下の手順になります。

1.  ローカルで開発サーバー起動: `npm run dev`
2.  コードを修正・確認
3.  本番用ビルド: `npm run build`
4.  デプロイ: `firebase deploy`
